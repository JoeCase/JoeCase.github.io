The select method

Select is a very efficient and great way to get a new array from an existing one. In effect it is a way to filter an array into a new one (it is like filter in javascript). I am sure you can already think of many ways to use this and probably have been looking for something just like this if you haven’t already stumbled upon it.

In essence, select will pass in each element of an array, pass it through a block of code that you provide, and then push it to a new array that the method then returns. The uses of this are pretty endless as you can imagine. If you have an array of data and want to pull certain things out of it, without messing with the original array, then this is a great method. Lets check out some examples.

num_array = [ 1, 56, 498, 3, 2, 79, 5, 6, 7, 45]

What if we wanted to fine all the numbers in that array that are even and just work with those? Let’s do this.

even_nums = num_array.select { |n| n % 2 == 0 }
p even_nums
p num_array
=> [56, 498, 2, 6]
=> [1, 56, 498, 3, 2, 79, 5, 6, 7, 45]

As you can see, we went through each element in the original array, and evaluated it on the condition n % 2 == 0. Each value that evaluated to true, was pushed to the new array, values evaluated to false did not. The original array was untouched.

Lets see some string examples…

names_array = [ 'Sarah', 'Tom', 'Joe', 'Alex' , 'Anita', 'Maria', 'Ron']

Perhaps we want to find all the people with names that have only three letters.

short_names = names_array.select { |n| n.length == 3 }
p short_names
=> ["Tom", "Joe", "Ron"]

or people with names that start with the letter A ...

a_names = names_array.select { |n| n[0].downcase == 'a'}
p a_names
# => ["Alex", "Anita"]

And all the while…

p names_array
=> ["Sarah", "Tom", "Joe", "Alex", "Anita", "Maria", "Ron"]

…the original array is untouched.

So as you can see, select is a very efficient and simple way to get a new array by filtering an existing one based on any condition want to setup.
