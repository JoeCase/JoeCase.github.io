Ruby Classes

Everything in Ruby is an object, an this is a great thing. The concept of an object can be a little abstract when dealing with a programing language, or computers in general, because when we think of an object, we usually think of something that we can see, touch and manipulate. And though we can’t really see or touch a Ruby object, we can manipulate it like a real object and thinking of it as a physical object is a great way to understand what exactly is going on under the hood.

When we look at objects in the real world, we notice that certain objects, they individually all belong together. Say lamps for example. We can see two lamps that are different in style and size and maybe even some functionality. But we know they are all lamps, because they share certain characteristics. All lamps turn on, all lamps turn off, all lamps need electricity and all lamps have a bulb that lights up a room. All lamps light. These shared characteristics make us know its a lamp and not a table. These shared characteristics are what let us know all the objects are related. This is what a class does in Ruby. It allows us to group objects that have shared characteristics. But what the most important thing it does is allow us to create as many objects from the same prototype as we want. A class can be thought of as a kind of blueprint or prototype. After we spawn the object that has all the basic shared characteristics we need, we can alter the children or instances, how we see fit.

Lets take say we wanted to make a group of airplane objects. We know all bikes have certain characteristics that make them different from cars. For instance, planes can fly. Planes have wings. And planes all takeoff and land. There are a lot of things different kinds of planes don’t share, such as color, type of engine and number of seats. If we wanted to make a bunch of planes, we would at least want to start with the most basic blueprint and then modify the special cases after that. Say we do this…

class Plane
attr_accessor :engine
attr_accessor :wings

def initialize
@wings = 2
@flight_status = :on_ground
@engine = :jet
end

def takeoff
@flight_status = :in_air
end

def land_it
@flight_status = :on_ground
end
end

With this we can start out and see that each place will two wings, a status of on the ground, and whatever kind of engine we start it with. It will also have the ability to take off and land. We can make a couple planes:

my_plane = Plane.new
your_plane = Plane.new

We both start with a jet engine and two wings. A great place so start. From here we can modify our plane, but at least we start with the basics, that all planes share. I could change the number of wings on my plane like this:

my_plane.wings = 4
p my_plane.wings
# => 4

From here we can make our planes as custom as we want. But we still know they are planes, and we could make a hundred new planes pretty quickly and not have to build the wings and engine every time.

So classes in Ruby are a great way to create a number of objects that share characteristics and behaviors.
